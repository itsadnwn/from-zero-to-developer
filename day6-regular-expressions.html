Regular Expressions: What I Wish I'd Known Earlier

16 July, 2025

Before I begin today's post, I just wanted to a share a little tiny win: I managed to upload this journey-blog onto GitHub! It took some reading and plenty of YouTube videos, but I've (kinda) figured it out and will post about what I've learned tomorrow!

Okay, now back to today's focus: Regular Expressions, also known as Regex.

I'll be honest - when I first encountered regular expressions, they looked like someone had fallen asleep on their keyboard. All those symbols and slashes seemed completely incomprehensible, so I took some time to dive in and understand it better. And after actually trying to use them, I've come to realize that they're one of those tools that... once you get it, you wonder how you ever lived without them.

The Lightbulb Moment

The breakthrough for me was understanding that regex isn't about memorizing cryptic symbols, it's also describing patterns. Instead of searching for "name@email.com" specifically, I could create a pattern that finds any email address. That shift from thinking about exact matches to pattern matching was everything.

I started with simple literal characters. The pattern cat matches the word "cat" - nothing fancy there. Stay with me though. But then I learned about the dot (.) which matches any single character, and suddenly c.t could match "cat", "cut", "cot", and so on.

That's when it clicked: these weren't random symbols but a logical system for describing what I wanted to find.

The Building Blocks

Once I got past the initial intimidation, I focused on the core concepts that do most of the heavy liftings:

Metacharacters (Special Characters)

Metacharacters are the fundamental special characters that have meaning beyond their literal value.

Character Description
. Matches any single character (except newline)
\ Escape acharacter - makes special characters literal. For example, \. would make the period literal.

Quantifiers

Quantifiers are a subset of metacharacters that specify how many times something should match.

Quantifier Description Example Matches
* 0 or more of preceding element colou*r color, colour, colouur
+ 1 or more of preceding element colou+r colour, colouur. (not color)
? 0 or 1 of preceding element colou?r color, colour
{n} Exactly n occurrences \d{3} Exactly 3 digits
{n, } n or more occurrences \d{3,} 3 or more digits
{n,m} Between n and m occurrences \d{3,5} 3, 4, or 5 digits

Anchors (Position Makers)

Anchors are position-based metacharacters that don't consume characters but match locations.

Anchor Description Example Matches
^ Start of line/string ^Hello "Hello" at the beginning of the line
$ End of line/string world$ "world" at the end of the line
\b Word boundary \bcat\b "cat" as a whole word, not in "catch"
\B Non-word boundary \Bcat\B "cat" inside words like "locate"
\A Start of the string (absolute) \AHello "Hello" at the very start of the string
\Z End of the string (absolute) world\Z "world" at the very end of the string

Character Classes

Character classes define sets of characters that match at a position.

Predefined Character Classes

Class Description Equivalent
\d Any digit [0-9]
\D Any non-digit [^0-9]
\w Word character [a-zA-Z0-9_]
\W Non-word character [^a-zA-Z0-9_]

Custom Character classes

Pattern Description Example Matches
[abc] Any of the characters a, b, or c [aeiou] Any vowel
[^abc] Any character except a, b, or c [^aeiou] Any consonant
[a-z] Any lowercase letter [a-z]+ Lowercase words
[A-Z] Any uppercase letter [A-Z]+ Uppercase words
[0-9] Any digit [0-9]{4} 4-digit number
[a-zA-Z] Any letter (upper or lower) [a-zA-Z]+ Any word
]a-zA-Z0-9] Any alphanumeric character [a-zA-Z0-9]+ Username format

HTML Form Regex Validation Example

Basic Personal Information

Must be 2-30 letters only
Pattern: [A-Za-z]{2,30}
Allows letters, spaces, hyphens, and apostrophes
Pattern: [A-Za-z\s\-']{2,50}
3-20 characters: letters, numbers, underscores only
Pattern: [a-zA-Z0-9_]{3,20}

Contact Information

Standard email format
Pattern: [a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,}$
Format: 123-456-7890
Pattern: [0-9]{3}-[0-9]{3}-[0-9]{4}
Flexible format: allows +, spaces, dashes, parentheses
Pattern: [\+]?[0-9\s\-\(\)]{10,15}
Must start with http:// or https://
Pattern: https?://[a-zA-Z0-9.-]+\.[a-zA-Z]{2,}

The Tools That Made Learning Easier

Online regex testers became my best friend. Being able to type a pattern and immediately see what it matches (or doesn't match) made learning so much faster than trying to debug in my head.

Another resource that really helped solidify my understanding was Corey Schafer's regex tutorial on YouTube. His clear explanations and practical examples made some of the more complex concepts click for me.
