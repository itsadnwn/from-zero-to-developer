CSS Box Model and Display: What I've Learned So Far

19 July, 2025

The box model was one of those concepts that made my brain hurt at first. But after wrestling with it for awhile, some things finally clicked. I wanted to share what I've learned so far - not as an expert (I'm far from it), but as someone who's still figuring it out alongside you.

The Big Picture: Everything is a Box

One thing that really helped me understand CSS was realizing that every single element on a webpage is essentially a rectangular box. Even circles and fancy shapes - they're all living inside invisible rectangles. One I accepted this, the box model started making more sense.

Longhand vs Shorthand: Finding My Balance

When I first learned CSS, I was intimidated by shorthand properties. So I stuck with longhand because it felt explicit and safe:

.my-box {
  margin-top: 10px;
  margin-right: 20px;
  margin-bottom: 15px;
  margin-left: 5px;

  padding-top: 12px;
  padding-right: 16px;
  padding-bottom: 12px;
  padding-left: 16px;
}

Now that I'm more comfortable, I realized shorthand wasn't as scary as I thought. Here's what I learned about the different patterns:

  • Four values: margin: 10px 20px 15px 5px (top, right, bottom, left - clockwise)
  • Three values: margin: 10px 20px 15px (top, horizontal, bottom)
  • Two values: margin: 10px 20px (vertical, horizontal)
  • One value: margin: 10px (all sides)

The Box

Content Box: The Heart of It All

The content box (img reference: in blue) is where your actual content lives - text, images, whatever. By default, when you set width and height, you're setting the size of this content box. Everything else (padding, border, margin) gets added on top.

Padding: The Breathing Room Inside

Padding was actually the easiest part for me to understand. It's like the cushioning inside a box - it pushes the content away from the edges. One thing to remember though: padding affects the total size of an element!

Margin: The Space Between Things

Margin create space around elements, and they have this weird thing called "margin collapse" that I'm still wrapping my head around. If you have 2 elements stacked vertically with margins, they don't add up - they collapse into the larger margin. Does that make sense? Let me give an example!

.box-one {
  margin-bottom: 30px;
}

.box-two {
  margin-top: 20px;
}

The space between these boxes isn't 50px - it's just 30px. Horizontal margins don't collapse though, only vertical ones!

Box-Sizing: The Game Changer

This property literally changed how I approach CSS. There are actually 3 different values for box-sizing, and understanding them made a huge difference in how I write CSS.

content-box (Default)

.content-box {
  box-sizing: content-box;
  width: 300px;
  padding: 20px;
  border: 5px solid blue;
  /* Total width: 350px (300 + 20 + 20 + 5 + 5) */
}

This is what I started with, and it's the default behaviour. The width and height you set only apply to the content area. Everything else gets added don top.

border-box (My New Best Friend)

.border-box {
  box-sizing: border-box;
  width: 300px;
  padding: 20px;
  border: 5px solid blue;
  /* Total width: exactly 300px! */
  /* Content area becomes: 250px (300 - 20 - 20 - 5 - 5) */
}

Once I discovered this, my life got so much easier. Now when I say something is 300px wide, it's actually 300px wide, including padding and border. The content area shrinks to accommodate them.

padding-box (The Rarely Used One)

.padding-box {
  box-sizing: padding-box;
  width: 300px;
  padding: 20px;
  border: 5px solid blue;
  /* Total width: 310px (300 + 5 + 5 for borders) */
  /* Width includes content + padding, but not border */
}

I've barely used this one. It includes padding in the width calculation but not the border. It's kind of a middle ground between the other two, but I haven't found a practical use case for it yet.

Display Properties: How Elements Behave

This is where things get really interesting. The display property basically tells an element how to behave in the layout.

Block Elements

.block-element {
  display: block;
  /* Takes full width, forces line breaks */
}

Block elements are like bullies - they take up the entire width and push everything else to new lines. Divs, headings, and paragraphs are block by default.

Inline Elements

.inline-element {
  display: inline;
  /* Flows with text, can't set width/height */
}

Inline elements play nice with text. They flow along with it, but you can't set their width or height. Spans and links are inline by default.

Inline-block: The Best of Both Worlds

.inline-block-element {
  display: inline-block;
  /* Flows with text BUT can have width/height */
}

This is my go-to for things like buttons or navigation items. They line up horizontally but still respect width and height.

CSS Reset: Starting with a Clean Slate

One thing that frustrated me early on was how different browsers have different styles. Here's a basic reset that I've learned:

* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

What I'm Still Learning

I'll be very honest - I still struggle with some aspects of CSS. Margin collapse still trips me up sometimes, and don't get me started on positioning and z-index (which I will talk about in the next post). But understanding the box model has made everything else feel more approachable.

The biggest lesson for me was to use browser dev tools constantly. Being able to see the box model visually in the inspector made everything click.
