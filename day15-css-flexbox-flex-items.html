CSS Flex Items Properties

21 July, 2025

CSS Flexbox has been something that left me overwhelmed, even until now. But after working with flex items for awhile, I've discovered that understanding these properties is like having a superpower for layout design. Let me share what I've learned about the six main flex item properties and when to use them.

What Are Flex Items?

Before diving into the properties, let me clarify what flex items are. When you set display: flex; on a container, all direct children of that container become flex items. These items can then be controlled using specific properties that only work on flex items.

The Flex Items Properties

flex-grow

This property controls how much a flex item should grow relative to other items in the container. I think of it as "how greedy is this item for extra space?"

flex-shrink

This property determines how much a flex item should shrink when there's not enough space.

  flex-shrink: 1;
  width: 200px;

  width: 300px;

  flex-shrink: 3;
  width: 500px;
}
flex-basis

This property sets the initial size of a flex item before free space is distributed. Not to be confused with width, because this is more flexible and it respects the flex direction.

All Values:
  • auto (default): Uses the item's width/height
  • Any length value (200px, 20em, 50%, etc)
  • 0: Item starts with no size

align-self

This property allows individual flex items to override the container's align-items setting. This was a game-changer when I needed one time to behave differently.

align-self: auto;

Inherits from the parent's align-items, which in this case, the parent's align-items is flex-end

This property controls the visual order of flex items without changing the HTML structure. This blew my mind when I first discovered it!

Wrapping Up

These flex item properties transformed how I approach layout design. I'm still trying to understand them, but with more practice, I'm sure I'll be able to get the hang of it. The key is to keep practicing and experimenting, and see how these properties interact with each other.

Flexbox is about flexibility. These properties are able to give me precise control over how that flexibility behaves. It's time to get comfortable with them!
