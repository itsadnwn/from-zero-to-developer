Git & GitHub: A Coder's Best Friend

17 July, 2025

Man, I wish someone had explained Git and GitHub to me like this when I first started learning all of these weeks ago. I spent way too much time being confused about what these things actually do and why everyone kept saying they were "essential". Now that I finally get it, I want to share what I've learned.

Here are the two videos that have really helped me understood Git and GitHub:

  1. Git Tutorial for Dummies by Nick White
  2. How to Use GitHub Pages in 2025 by The Common Coder

The Lightbulb Moment about Command Line

I admit I avoided the command line for the past few weeks because it looked scary. All those black screens with white texts - it was all too confusing and it felt like hacker movie stuff. But then I was given this analogy that changed everything.

Think about ordering food. You know how some apps make you click through 50 different screens?

Open the app → Browse menus → Click appetizers → Scroll through 20 options → Add to cart → Go back → Click entrees → Scroll through 30 more options → Add to cart → Checkout → Fill out forms → Submit

It's exhausting... Even typing that out, I'm exhausted already.

When I want to save my code to GitHub now, instead of clicking through VS Code's menus:

Source Control → Click the + next to each file → Type message → Commit → Sync → Wait

Now I just type:

  1. git add .
  2. git commit -m 'what was done'
  3. git push

And boom! Done in 3 seconds. The command line isn't trying to be intimidating. It's just the fastest way to tell your computer exactly what you want.

What Git Actually Is (And Why I Need It)

I used to think Git was just for real programmers working on huge projects. Turns out, I needed it from day one and I will always need it. It's like my best friend now.

Git is like a time machine for your code. Every time I save my work (called a 'commit'), it takes a snapshot of everything. Broke my code? I can go back to yesterday's version in seconds. Want to try something risky? I can create a "branch" and experiment without breaking up my main code.

It's like having a safety net for my code. If I mess up, I can just roll back to the last working version. No more panicking about losing hours of work.

Why I Finally Joined GitHub

I'll be honest - I was intimated by GitHub at first. It felt like this exclusive club where only real developers hung out. But I now realize that I was missing out on so much.

GitHub is basically social media for programmers, but way more useful than Instagram.

  • It's my portfolio now. Okay, it's still empty now. But it'll be my portfolio for when I apply for jobs in future because employers would want to see my actual code. GitHub is where I can show what I can build. It's like a public workshop where people can see the quality of my work.
  • It's where I learn. GitHub has millions of projects I can study. Found a cool website? The code is probably on GitHub and I can study it, learn from it, and try to create it on my own.
  • It's my backup. I haven't had any disasters yet (knock on wood), and while I do save my files on Google Drive as well, it is nice to know that my code is safe in the cloud of GitHub. This is a peace of mind that every developer desires (I would like to think so because I'm not a developer yet and I already feel that way).
  • It's where I can contribute. Even though right now I'm just contributing this self-learning journey blog and there's probably not much anyone can benefit from at this moment, I know that I will be able to contribute to better codes and projects in future as I progress.

GitHub Pages: My Free Website (Wow!)

I had no idea GitHub gives you free website hosting. Like, actually free. Every repository can become a live website. Here's how I learned to set it up:

  1. I go to one of my repositories on GitHub, click on "Settings" at the top.
  2. Scroll down to "Pages" in the sidebar.
  3. Under "Source", I select "Deploy from a branch", and choose "master" as my branch.
  4. Then click "Save".

That's it. GitHub gives me a URL and my blog now is live.

My Honest Take

I'm still very new to all these, and I'm sure there are still a lot for me to learn, but Git and GitHub don't feel like this huge mountain anymore. They're what let me actually feel like I'm taking a step closer to becoming a developer.

If you're in the same boat as me, just start. Make a GitHub account, learn Git. Create something simple and put it online. We'll figure out the rest as we go.

I'm learning that everyone starts somewhere, and the developers around the world were beginners once, too.
