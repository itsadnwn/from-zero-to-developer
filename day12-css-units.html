CSS Units: What I've Learned About Absolute and Relative Units

20 July, 2025

Diving back into CSS Units because I used to be confused about all the different units that can be used. After spending some time to read through different explanations, and watching Colt Steele's CSS Units Video, I finally understand when to use what. Let me share what I've learned about CSS units and hopefully it'll help you, too!

CSS units fall into two camps: absolute units (which stay the same size no matter what) and relative units (which change based on other things). When I first started, frankly I just used pixels for everything because it seemed simpler. But really, it wasn't always the best choice!

CSS Units
  Description Example When to Use
Absolute Units
Pixels (px) One pixel equals one dot on the screen, and it stays the same size regardless of anything else. But it's really not recommended for responsive websites. .button {
  width: 200px;
  height: 40px;
  font-size: 16px;
}
  • Border widths (like border: 1px solid black;)
  • Small decorative elements
  • When I need something to be exactly the same size everywhere
Points (pt) One point equals 1/72 of an inch, which comes from the printing world. Not really for screen designs. *Ditching the example because it's not really relevant*
  • Pretty much only for print styles
Relative Units
Em (em) An em is relative to the font size of the element it's used on. So, if an element has a font size of 16px, then 1em=16px for that element. .parent {
  font-size: 20px;
}

.child {
  font-size: 1.5em; /*This becomes 30px (20px * 1.5) */
  margin: 2em; /* This becomes 60px (30px * 2) */
}
  • Padding and margins that should scale with text
  • When I want elements to grow/shrink with their font size
Root Em (rem) Em's simpler cousin. Instead of being relative to the current element's font size, rem is always relative to the root element's font size (usually 16px). html {
  font-size: 16px;
}

.card {
  font-size: 1.2rem; /* Always 19.2px */
  padding: 1rem; /* Always 16px */
  margin: 2rem; /* Always 32px */
}
  • Font sizes
  • Consistent spacing through the shite
  • Pretty much everywhere I used to use pixels lol
Percentages (%) Percentages are relative to the parent element. This seemed straightforward until I realized it depends on which property you're using. .container {
  width: 800px;
  height: 400px;
}

.box {
  width: 50%; /*This becomes 400px (50px of 800px) */
  height: 25%; /* This becomes 100px (25% of 400px) */
  font-size: 150%; /* 150% of parent's font size */
  padding: 10%; /* 10% of parent's WIDTH (80%), not height! */
}
  • Making responsive layouts
  • Creating flexible containers
  • Grid and flexbox layouts
Viewport Units (vw, vh, vmin, vmax) Relative to the viewport (the browser window):
  • vw: 1% of viewport width
  • vh: 1% of viewport height
  • vmin: 1% of the smaller dimension
  • vmax: 1% of the larger dimension
 .hero-section {
  height: 100vh; /* Always fills the full height of the screen */
}

.responsive-text {
  font-size: 4vw; /* Scales with screen width */
}
  • Full-screen sections
  • Responsive typography
  • Elements that should scale with the screen

Hope this helps to avoid any confusion that you might have! CSS units can be tricky, and I'm still trying to get the hang of them as well. As tricky as they can be, they're actually pretty powerful tools for creating flexible, responsive designs. And I can't wait to experiment with them in my future projects!
