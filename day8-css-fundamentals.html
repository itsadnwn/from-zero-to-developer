My First Steps with CSS: What I Learned about Selectors, Specificity, and Properties

18 July, 2025

When I first heard about CSS, I thought it was just about making websites look pretty. Change some colors here, adjust some fonts there, and boom - you're a web designer, right? Well... I was wrong. After diving in and trying it out in practice, I quickly realized there's way more to it than I expected.

I'm still pretty new to all these, and I haven't really tried to build and beautify anything yet. But I just wanted to share some of the things I've learned so far.

CSS Selectors: How Do I Even Target Things?

When I started, I was a little overwhelmed by all the different ways to select elements in CSS. Like there are SO many options, and I still had no idea when to use what.

The basics were okay. I could handle p for paragraphs, h1 for headings. Classes with .my-class made sense, and IDs with #my-id are supposed to be unique.

But then I started seeing things like .container > .item:first-child and my brain just went out the window. Until it was explained to me, then I realized it's not that scary.

The > means "direct child of" - so .container > .item means "find items that are direct children of containers, not grandchildren or great-grandchildren". And :first-child is just a way to say "the first one in the group.

The colon ones (:first-child, :hover) are called pseudo-selectors, and they're basically ways to target elements in special situations. Think of them as adding extra conditions to your regular selectors.

Like :hover - that's for when someone puts their mouse over something. So button:hover means "style the button, but only when someone's hovering over it".

And :first-child is for targeting the first item in a group. If you have a list of 5 items, li:first-child will only style the very first item in that list.

One thing that blew my mind was attribute selectors. Instead of adding classes to every single input field, I could just write input[type="email"] to target all email inputs! It feels like cheating, but in a good way.

CSS Specificity

CSS has this whole system for deciding which styles "win" when there are conflicts. It's called specificity, and it's basically a scoring system. More specific selectors beat less specific ones.

Here's what I learned (and I'm definitely oversimplifying):

  • Inline styles (like style="color: red;" beat everything
  • IDs beat classes
  • Classes beat element tags
  • When scores are tied, the last one in the CSS file wins

CSS Properties

The sheer number of properties is honestly overwhelming. I mean, just take a look at this! Every time I think I've learned the main ones, I discover a whole new category. Layout properties, typography properties, animation properties, grid properties, flexbox properties (where I struggled) - it never ends.

But here's what I've learned: you don't need to memorize them all. You just need to know the basics, and then look up the rest when you need them.

My Current Mindset Towards CSS

These are the things that I will be reminding myself of as I continue to refresh on and learn about CSS:

  1. It's okay to be confused. CSS can be counter-intuitive sometimes. I just need to be patient and gentle with myself, I'm still learning.
  2. Start simple. Don't try to build complex layouts right away. Master the basics first.
  3. The browser dev tools are my friends. I can inspect elements and see exactly what styles are being applied. This was a huge game-changer by the way.
  4. Practice, practice, practice. Reading about CSS is one thing, but actually building things is how it really clicks.
  5. Don't memorize everything. Learn the fundamentals first, then look up the rest as needed.

Where I'm Learning

I've been working through Colt Steele's Web Development Course on YouTube, and it's been amazing for actually understanding these concepts through hands-on projects. He explains things in a way that makes sense to beginners.

When I need to look up specific properties or understand how something works, the MDN CSS Reference has been incredibly helpful. It's super detailed, but it's accurate and comprehensive.
